The MEAN App Tutorial

The Web App on the MEAN Stack


Encounter unknown terminology? Check our Glossary.
Prerequisites: Complete the One MEAN Tutorial before continuing this section.

Reminder: The lines that look like this are to be entered into the command prompt. Be patient, some commands may take time to complete, that is absolutely normal.

1. Remove the current server.js file:

sudo apt update -y

2. Update the server.js file:

sudo wget -O server.js https://raw.githubusercontent.com/ngocdai94/meanStack/main/server.js


Note: So this one is a little complicated, the mean stack comes with a default server.js file which we need to replace with a basic web-server from a MEAN repo on github hosting a basic web-server server.js file. So we essentially swap the old server.js file for the new one.

3. Access the public directory, the location of your website content:

cd public


4. Copy the website content from the github repo to your public directory:

git clone https://github.com/Titane73/it115-a5-g2-app.git

5. Change to the MEAN directory:

cd ..

6. Run Gulp:

sudo gulp

7. Go to your Google Cloud Platform website

This is where you made your VM instance in the MEAN stack tutorial. There are ways to get the External IP to properly display in your preferred web browser:

  • The first method is to look for your instance and hover over the External IP number to reveal a ‘copy to clipboard’ icon next to it. Click on the icon to copy your website link into your clipboard. Paste the copied website link into the URL of a new tab.
  • The second method is to look for your instance, hover directly over the External IP until it is highlighted, and then left-click (this will open a new tab).

If you will a screen that shows that a connection cannot be reached, don’t fret. Double-click the IP address in the URl until the whole URL is visible. It should have 'https://" on the front. Delete the ‘s’ character and reload the webpage. You should be able to see the website content.

Congratulations!
